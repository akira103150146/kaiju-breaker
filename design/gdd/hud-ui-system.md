## A. 概覽 (Overview)

HUD・UI 系統是殲獸戰機所有玩家介面的唯一設計規範。它橫跨兩種情境：**戰鬥中（In-Combat）**與**戰鬥外元介面（Meta Screens）**。

戰鬥中，HUD 的首要職責是**「讓彈幕讀得懂」**——任何介面元素只要干擾到子彈可讀性，就是設計失敗，無論其資訊多麼有用。本系統以「需要知道時才顯示、不需要時靜止或隱藏」為原則，把有限的螢幕空間全數還給彈幕閃避。

戰鬥外，介面的首要職責是**「讓選擇輕鬆但有重量」**——Loadout 選擇、武器升級、難度設定都是影響接下來一局的決策，介面必須提供充分資訊而不製造選擇疲勞。

**設計鐵則（不可妥協）：**

1. 「**彈幕永遠讀得懂**」— HUD 元素不得遮蔽敵彈外框或玩家判定點（1 像素白點）
2. 「**冷色＝你，暖色＝威脅**」— 玩家陣營 UI 用冷色；敵彈、危險警示用暖色（唯一例外：素材軌道球以青綠 #62F0D8 表達「收獲」，此為 game-feel.md 已定義的色彩，不得更改）
3. 「**不重複 game-feel 已做的事**」— `game-feel.md` 已在部位上渲染 SOFTENED 橙紅光暈（#FF6600）；HUD 只加血條數值，不再疊加相同的狀態文字
4. 「**難度是門**」— 任何難度相關 UI 不鎖內容，只描述壓力差異

## B. 玩家幻想 (Player Fantasy)

**目標 MDA 美學**：感官愉悅（Sensation）＋ 表達（Expression）＋ 挑戰（Challenge）

**戰鬥中**：HUD 幾乎透明——不是因為資訊少，而是因為資訊**在對的地方、在對的時機**。副武器彈匣快耗盡時，右下角的彈頭圖示自然引起注意；部位進入 SOFTENED 時，光暈在怪獸身上脈動，玩家無需移開視線確認狀態。HUD 是戰場的延伸，而不是覆蓋在戰場上的另一個圖層。

**戰鬥外**：每次進入裝備畫面，玩家都清楚知道自己在決定什麼：「我要用那把雷射，搭上這把飛彈，去打那隻怪，為了升這個武器的 Tier-3。」這是養成遊戲的滿足感，不是資訊過載的焦慮感。

## C. 資訊架構 (Information Architecture)

### C.1 資訊層級（全域）

| 層級 | 類型 | 範例 | 可見條件 |
|------|------|------|---------|
| **P0 永駐（Always-On）** | 不可遮蔽 | 玩家判定點 1px 白點 | 永遠 |
| **P1 戰術（Tactical）** | 需即時反應 | 副武器彈匣、換彈條、L3 蓄力條 | 戰鬥中持續顯示 |
| **P2 狀態（Status）** | 精確數值 | 部位 HEAT 條、BREAK 條 | 部位 ALIVE 期間 |
| **P3 進度（Progress）** | 場景脈絡 | 素材計數、分數、Boss 整體條 | 戰鬥中持續 |
| **P4 情境（Contextual）** | 偶發提示 | 一次性新手提示、「SOFTENED!」浮動文字 | 事件觸發；自動淡出 |

**規則**：P0 z-order 置於最頂層，任何閃光或遮罩**均不得**覆蓋。P4 在無障礙設定中可由玩家關閉。

- **播放區中央**：無固定 HUD 元素；部位血條以 **World-Space（世界座標）** 方式貼附在 Boss 部位上

## D. 戰鬥中 HUD (In-Combat HUD)

### D.1 主武器槽（Primary Weapon Slot）— C 區左下

顯示當前裝備的雷射系武器（L1–L4）。

| 元素 | 說明 | 更新時機 |
|------|------|---------|
| **武器圖示** | 16×16 像素圖示 | 換裝時即時切換 |
| **武器名稱** | 中文簡稱（≤ 4 字），英文縮寫在括號內 | 換裝時即時切換 |
| **Tier 徽章（Tier Badge）** | T0 / T1 / T2 / T3 小標籤，貼圖示右上角 | 升級後更新 |
| **L3 蓄力條（Charge Bar）** | **僅 L3 波動砲（Wave Cannon）裝備時顯示**；水平進度條 0→1.5s | 每幀（Hold 期間） |
| **L3 冷卻遮罩（Cooldown Mask）** | 蓄力釋放後 2s 冷卻，灰色遮罩從右向左收縮 | 每幀（CD 期間） |

**L3 蓄力條行為細節：**

| 狀態 | 視覺 |
|------|------|
| 待機（未按下） | 空條輪廓，無填充 |
| 充能中（Hold） | 冷色→暖黃，從左向右填充 |
| 就緒（≥ 90%，旋鈕 `l3_charge_bar_flash_pct`） | 條體閃爍，提示「現在可以釋放」 |
| 冷卻中（釋放後 2s） | 灰色遮罩由右向左收縮，露出進度條 |
| 非 L3 武器時 | 條完全隱藏，不占空間 |

**設計理由**：L1/L2/L4 為持續輸出，無需額外狀態條。L3 雙模式（Tap / Hold）是其核心設計，充能狀態直接影響每次按鍵決策——屬 P1 資訊，必須常駐可見。

### D.2 副武器槽（Secondary Weapon Slot）— D 區右下

顯示當前裝備的飛彈系武器（M1–M4）。

| 元素 | 說明 | 更新時機 |
|------|------|---------|
| **武器圖示** | 16×16 像素圖示 | 換裝時即時切換 |
| **武器名稱** | 中文簡稱 + Tier 徽章 | 換裝/升級時更新 |
| **彈頭圖示列（Ammo Pips）** | N 個小圖示，滿格＝實心，已用＝空心 | 每次射擊後即時 |
| **換彈進度條（Reload Bar）** | 換彈期間橫向填充；填滿後彈頭圖示全部恢復實心 | 換彈期間每幀 |

**各武器彈匣規格：**

| 武器 | 彈匣 | Pip 數 | 圖示建議 |
|------|------|-------|---------|
| M1 追蹤飛彈（Homing Missile） | 6 枚 | 6 | ●●●●●● |
| M2 蜂群飛彈（Swarm Launcher） | 8 枚（Tier-3 升為 12） | 8（Tier-3 見 J.2） | ●●●●●●●● |
| M3 穿甲魚雷（AP Torpedo） | 3 枚 | 3（大圖示） | ◆◆◆ |

**彈匣低彈警告**：剩餘 ≤ `ammo_pip_low_threshold`（預設 1）時，彈頭圖示以 1 Hz 閃爍。不依賴顏色，形狀閃爍即為警告信號（色盲安全）。

**M4 落點指示器（World-Space）**：武器系統提供落點座標；本系統在播放區渲染半透明細環（冷色），隨玩家橫向移動而移動。Z-order 置於子彈層**之下**，不干擾敵彈可讀性。**此元素不在 C/D 區 HUD 槽內顯示**；為世界座標元素，歸類至 P2。

### D.3 素材計數器（Material Counter）— B 區右上

戰鬥中以**最小化方式**呈現，不打斷戰鬥注意力。

| 元素 | 說明 |
|------|------|
| **通用碎片（Common Shard）** | 圖示 + 數字（本場累積量） |
| **核心總數（Core Count）** | 本場已得核心數量（不分類型；明細留給結算畫面） |
| **精魄提示（Essence Flash）** | 僅全破壞結算觸發後顯示 1 次（精魄入袋動畫），平時不佔空間 |

**素材飛入動畫（配合 game-feel.md D.4）**：材料軌道球（青綠 #62F0D8）抵達計數器時，計數器彈跳（`material_counter_bounce_duration_s = 0.3s`）＋「+N」浮動文字淡入淡出。此動畫由 game-feel 系統觸發，本系統接收計數更新事件後執行 UI 數字跳動。

**設計理由**：具體核心類型（甲殼 / 四肢 / 能量）在戰鬥中不影響即時決策；區分核心類型的認知負擔超過其戰鬥中的價值。玩家只需知道「我在獲得素材」——滿足「破壞即獎勵」的感知，明細交給結算畫面。

### D.4 關卡進度（Stage Progress）— A 區左上

| 元素 | 說明 |
|------|------|
| **分數（Score）** | 右對齊 6 位數，固定格式 000000 |
| **關卡標識** | 例「Stage 1-2」|
| **Boss 整體條（Boss HP Bar）** | 僅 Boss 戰段顯示；細長橫條，由 BOSS_CORE 的 B_current / B_max 驅動 |

**Boss 整體條設計意圖**：只追蹤 BOSS_CORE 的破甲進度，**不**聚合所有部位。這保留「只打核心 vs 全部位破壞」的策略張力——玩家看整體條無法知道其他部位狀態，必須主動觀察世界座標血條來評估素材收益。

### D.5 玩家判定點（Hitbox Indicator）— 永駐 P0

玩家飛機中心有一個 1 像素的白色高亮點：
- **永不消失**，不受任何遮罩、閃光、慢動作影響
- Z-order 位於渲染堆疊絕對頂層，高於全白閃光覆蓋層
- 與 `game-feel.md` 的 `flash_max_alpha = 0.85`（非 1.0）協調——確保閃光下仍可見，此為硬性約束

## E. 部位狀態視覺化 (Part-State Visualization)

### E.1 設計決定：世界座標貼附（World-Space On-Part）

**決定**：HEAT 條（熱量槽）與 BREAK 條（破甲槽）以微型世界座標血條形式，**貼附在各部位 Sprite 框的上下方**，隨部位移動。**不**在 HUD 角落設立獨立的部位狀態面板。

**理由：**
1. **注意力管理（Attention Management）**：Boss 占據螢幕大部分。玩家視線本就在 Boss 上。把資訊帶到玩家視線所在，而不是要求視線去角落搜索——Fitts's Law 的基本原則，也是「彈幕永遠讀得懂」的直接落實。
2. **與 game-feel.md 協調不衝突**：game-feel.md 已在部位上渲染 SOFTENED 橙紅光暈 + 脈動（世界座標）。世界座標血條是這套視覺語言的量化延伸，是補充而非重複；角落 HUD 面板才是真正的重複。
3. **彈幕可讀性**：固定角落面板佔據可能有子彈飛過的螢幕區域；世界座標血條與部位同步移動，設計師可在 Boss 設計時確保血條不落在高密度彈道區。
4. **手機相容性**：固定角落面板在手機小螢幕上更容易被拇指覆蓋，且與世界坐標元素的視距不一致。

### E.2 部位血條規格

每個 ALIVE 部位在其 Sprite 框外渲染兩條薄型像素血條：

| 血條 | 位置 | 填充方向 | 色彩語義 | 數據來源 |
|------|------|---------|---------|---------|
| **HEAT 條（熱量條）** | 部位框**上方** `part_bar_offset_top_px`（預設 2px）間距 | 左→右，隨 H_current/H_max 填充 | 暖橙色為主（越接近 SOFTENED 閾值，橙色越飽和） | kaiju-part-system H_current |
| **BREAK 條（破甲條）** | 部位框**下方** `part_bar_offset_bottom_px`（預設 2px）間距 | 左→右，隨 B_current/B_max 填充 | 深藍→白色（「白色＝快要爆了」的直覺感） | kaiju-part-system B_current |

**ARMORED 部位額外視覺層：**

- **ARMOR_INTACT**：BREAK 條上疊半透明藍灰遮罩（`armor_mask_opacity = 0.65`），且 BREAK 條讀數凍結顯示（B_fill = 0 時對應 game-feel.md 的「彈頭偏轉」邏輯）
- **ARMOR_STRIPPED + STAGGERED**：遮罩消失，BREAK 條周圍出現 2px **細白外框**（= game-feel.md C.4 定義的「弱點框」），外框以倒計時視覺消退（`stagger_duration` 秒歸零）

**所有狀態組合的視覺協調總覽：**

| 部位狀態 | HEAT 條 | BREAK 條 | game-feel.md 效果（非本系統） |
|---------|---------|---------|---------------------------|
| INTACT（正常）| 灰→橙漸進填充 | 藍→白漸進填充 | 無 |
| SOFTENED | 滿格橙脈動（同步 2 Hz）| 藍→白漸進 | **橙紅光暈 #FF6600 + sfxSoften** |
| STAGGERED（普通/Core 部位）| 正常 | 白×1.5 速率填充 | 白色閃爍 + 冷藍火花 |
| ARMORED + ARMOR_INTACT | 正常 | 凍結（藍灰遮罩）| 無 |
| ARMORED + ARMOR_STRIPPED | 正常 | 白×1.5 速率填充 + 細白外框 | **弱點框 + sfxArmorStrip** |
| SOFTENED + STAGGERED | 滿格橙脈動 | 白×1.5 速率 + 外框（如適用）| 光暈 + 弱點框（如適用） |
| BROKEN | **不顯示** | **不顯示** | **爆炸碎片 + 素材軌道球** |

**分工聲明**：game-feel.md 擁有顏色偏移、光暈、閃光、粒子、音效；本系統擁有血條填充量、位置佈局、ARMORED 遮罩、倒計時外框。兩系統合力構成完整可讀性，無重疊衝突。

### E.3 血條尺寸規格

| 規格 | 數值 |
|------|------|
| 血條高度 | `part_bar_height_px = 3`（像素） |
| 血條寬度 | 等於部位 Sprite 寬度（貼合邊框） |
| 手機最小高度 | 縮放後實體高度 ≥ 6 螢幕像素（防止過細不可見） |
| 最大部位數上限 | 後期 Boss 最多 8 部位（見 J.1 邊界情況）|

## I. 無障礙（Accessibility）

### I.1 文字縮放（Text Scaling）

- 設定選單提供 100% / 125% / 150% 三段縮放
- 影響範圍：Meta Screen 說明文字、武器卡片說明、升級費用顯示、難度卡片說明
- **不影響範圍**：HUD 血條、武器圖示、彈頭 pip（這些是功能性像素元素，縮放已由全域 UI 縮放倍數控制）

### I.2 色盲安全設計（Colorblind-Safe Design）

本遊戲視覺哲學（冷色＝你，暖色＝威脅）高度依賴色彩。單靠顏色無法完全區分時，提供冗餘的非色彩線索：

| 情境 | 色彩線索 | 非色彩冗餘線索（必須實作）|
|------|---------|----------------------|
| 敵彈 vs 玩家彈藥 | 暖橙 vs 冷藍 | **形狀差異**（敵彈為圓點；雷射為細線；飛彈為橢圓形）|
| SOFTENED 部位 | 橙紅色調 + 光暈 | **2 Hz 脈動閃爍節律**（形狀可識別）+ **sfxSoften 音效**（可單靠音效感知）|
| ARMOR_STRIPPED 弱點 | 白色框 | **倒計時框消退動畫**（時間維度線索，無需辨色）|
| 素材軌道球 | 青綠 #62F0D8 | **拋物線飛行軌跡**（區別於直線 / 擴散的敵彈）|
| 彈匣低彈警告 | 彈頭圖示顏色偏暗 | **1 Hz 閃爍動畫**（節律線索，不依賴顏色）|

**色盲替代方案（設定選單一級選項）：**
- 預設（冷暖對比）
- 藍黃對比模式（Deuteranopia / Protanopia 友善）
- 形狀優先模式（所有關鍵狀態增加形狀圖示，大幅減少色彩依賴）

具體替代調色盤由美術總監（art-director）依本需求確認實作；本文件定義功能需求，不指定最終 hex 值（已定義色彩如 #FF6600、#62F0D8 由相應 GDD 確立，本系統繼承）。

### I.3 減少動態協調（Reduce-Motion — 配合 game-feel.md H.1）

Reduce-Motion 設定由本系統提供統一開關，廣播 `reduce_motion_changed` 事件，game-feel 系統訂閱並調整旋鈕。

| 效果 | Reduce-Motion 下（本系統責任） |
|------|-------------------------------|
| 素材飛入軌道球動畫 | 停用；計數器直接數字跳動 |
| 結算素材統計滾動動畫 | 縮短至 0.2s 淡入，不滾動 |
| 難度卡片 / 武器卡片進場動畫 | 停用，直接顯示 |
| 武器換裝圖示切換特效 | 停用，直接替換 |
| HUD 換彈進度條 | **保留**（功能性，非裝飾）|
| SOFTENED 脈動光暈 | 由 game-feel.md 控制（非本系統職責）|

### I.4 字幕（Subtitles）

MVP 階段無語音對白，字幕系統暫不實作。若後期加入語音（劇情夢想層），本 GDD 需更新，字幕作為 P4 資訊層加入。

### I.5 無障礙設定入口位置

無障礙設定（色盲模式、Reduce-Motion、文字縮放）放在主選單**一級選項**（不需要多層進入），表示對無障礙的承諾。

## J. 邊界情況（Edge Cases）

### J.1 後期 Boss 有 8 個部位，血條過密

**情況**：最多 8 個可破壞部位各有 2 條血條 = 最多 16 條世界座標條同時存在，可能重疊。

**處理**：
- 血條寬度 = 各自部位寬度（不越界），因此水平方向不重疊
- 垂直方向：Boss 設計師須確保相鄰部位的垂直間距 ≥ `part_bar_height_px × 2 + part_bar_offset_top_px × 2 + 2px`（≈ 12px），確保 HEAT 條和 BREAK 條不重疊
- 此約束作為 Boss 設計評審清單的一條（由 level-designer 負責）
- game-feel.md 的 SOFTENED 光暈仍完整顯示（光暈由部位 Sprite 觸發，不依賴血條位置）

### J.2 M2 蜂群飛彈 Tier-3 彈匣升至 12 枚

**情況**：M2 Tier-3 彈匣從 8→12 枚（分 2 次 6 枚連發）；8 個 Pip 圖示不足。

**處理**：M2 Tier-3 升級後，Pip 模式切換為「N/12」數字顯示（`m2_t3_use_numeric_display = true`，預設開啟），取代小圖示列。升至 Tier-3 時 UI 說明文字提示玩家顯示模式變更。

### J.3 拾取武器莢艙時 HUD 即時切換

**情況**：玩家拾取武器莢艙，舊武器的彈匣/換彈進度需清除，新武器狀態需立即顯示。

**處理**：
- 圖示、名稱、Tier 徽章：即時替換，無動畫延遲（確保玩家知道「現在手上是什麼武器」）
- 彈匣：重置為新武器的滿格狀態
- 換彈條：若舊武器正在換彈，立即清除；新武器不繼承換彈狀態
- L3 蓄力條：從 L3 切到其他主武器時立即隱藏；切換到 L3 時蓄力條從 0 開始

### J.4 輪進行中開啟設定選單（Pause Menu）

**情況**：玩家暫停遊戲並更改 Reduce-Motion / 色盲模式。

**處理**：
- Reduce-Motion / 色盲模式可即時切換並立即生效（無需重啟）
- 難度選項在暫停選單顯示為灰化，附說明「下一輪可更改」（繼承 `difficulty-system.md` E.1）
- 返回遊戲後，已改變的設定從下一幀起即時生效

### J.5 Boss 死亡後血條清理

**情況**：`on_boss_core_break` 發出後，本系統的血條需要適當清理。

**處理**：收到 `on_boss_core_break` 後：
1. 所有部位血條停止更新（忽略後續命中事件的 HUD 更新）
2. 隨 game-feel.md 的爆炸碎片粒子序列，血條伴隨部位死亡動畫消失（不突兀保留在螢幕上）
3. 勝利結算序列啟動後，進入結算畫面前血條全清除

### J.6 MVP 配置（僅 2 武器）的 Loadout 畫面

**情況**：MVP 僅有 L1 + M1，其餘 6 個武器格子需要顯示處理。

**處理**：未解鎖的武器格子顯示：鎖定圖示 + 「更多武器開發中」（MVP 版本），或留空（取決於導演決定）。升級畫面不顯示 Tier-3 預覽（MVP 不開放 Tier-3，由 `material-economy.md` G.4 `enabled_weapon_tiers` 控制）。

## M. 驗收標準（Acceptance Criteria）

### M.1 彈幕可讀性不受 HUD 干擾（體驗性 — **Alpha 阻斷**）

- [ ] D4 惡夢難度下，HUD 所有元素（血條、武器槽、計數器）不遮蔽任何正在飛行的敵彈外框：5 人用戶截圖測試，辨識率 ≥ 70%（繼承 `difficulty-system.md` H.7）
- [ ] 玩家判定點（1px 白點）在部位破壞全白閃光（flash=0.92）期間仍可在截圖中識別（z-order 驗證）
- [ ] 世界座標部位血條在 Boss 高速移動段不發生「血條飛出部位邊界」錯位（目視整合測試）

### M.2 SOFTENED 狀態在 ≤ 0.5s 內可讀（體驗性 — **Alpha 阻斷**）

- [ ] 繼承 `kaiju-part-system.md` H.2 / `weapon-system.md` H.5 / `game-feel.md` I.1：SOFTENED 視覺（光暈 + HEAT 條橙色脈動）必須在 0.5 秒內可被感知，5 人測試 ≥ 80% 辨識率
- [ ] HEAT 條在 `on_part_softened` 收到的**同幀**切換為滿格橙色脈動顯示，無跨幀延遲
- [ ] `softened_heat_bar_pulse_hz` 與 `kaiju-part-system.md` G.3 `softened_pulse_frequency_hz` 完全一致（自動化設定讀取測試）
- [ ] SOFTENED 退出（`on_part_softened_exit`）時，HEAT 條同幀停止脈動並回到正常進度顯示

### M.3 副武器彈匣狀態正確（功能性 — 阻斷）

- [ ] 副武器射擊後，對應 Pip 即時從實心→空心（無幀延遲）
- [ ] 彈匣耗盡後，換彈條立即開始填充
- [ ] 換彈完成後，所有 Pip 恢復實心，換彈條消失
- [ ] 拾取同類型武器莢艙：彈匣立即補滿，換彈進度清零
- [ ] 自動化測試：`tests/unit/ui/secondary_ammo_display_test.[ext]`，覆蓋 4 種副武器的彈匣狀態機（射擊 → 耗盡 → 換彈 → 完成 → 拾取替換）

### M.4 L3 蓄力條正確（功能性）

- [ ] 裝備 L3 時蓄力條顯示；裝備 L1/L2/L4 時蓄力條完全隱藏（不占空間）
- [ ] 按住觸發後，蓄力條從 0 填充至 100% 所需時間與 `l3_charge_time`（1.5s）一致（±1 幀容差）
- [ ] 充能 ≥ `l3_charge_bar_flash_pct`（90%）時，閃爍動畫啟動
- [ ] 釋放震波後，2s 冷卻遮罩正確顯示並在 `l3_charge_cooldown` 結束時消失

### M.5 升級畫面費用顯示正確（功能性 — 阻斷）

- [ ] 素材不足時，對應行顯示紅色 ✗ + 升級按鈕灰化（disabled）
- [ ] 完成狩獵返回升級畫面：庫存數字即時同步（不需手動刷新）
- [ ] Tier-3 模糊預覽：Tier < 3 時始終模糊；升至 Tier-3 後模糊完全消失，顯示完整說明
- [ ] 核心不足時，「建議狩獵」提示正確指向對應巨獸名稱與核心類型

### M.6 難度 UI 行為（功能性）

- [ ] 首次啟動：預選 D1（`default_difficulty_on_first_launch = D1` 生效）
- [ ] 輪結束返回：預填上次選擇難度（`remember_last_difficulty = true` 生效）
- [ ] 輪進行中暫停：難度選項灰化且附「下一輪可更改」說明，無法互動
- [ ] 自動化測試：`tests/unit/ui/difficulty_ui_state_test.[ext]`，覆蓋首次啟動 / 輪結束 / 輪中途三種狀態

### M.7 跨平台 HUD 安全區（功能性）

- [ ] **手機 Portrait**：所有固定 HUD 元素位於螢幕頂部 15% 與底部 20% 的 UI 條內；中間播放區無固定元素
- [ ] **手機**：副武器按鈕觸控目標 ≥ 44×44 dp（WCAG 2.1 AA）
- [ ] **PC 4K（3840×2160）**：UI 元素以整數倍縮放渲染，像素邊緣銳利（截圖確認，無模糊像素）

### M.8 無障礙功能（功能性 — Advisory）

- [ ] 文字縮放 150% 時，所有 Meta Screen 文字不截斷、不重疊
- [ ] 色盲替代色彩方案啟用後：SOFTENED 狀態可透過脈動節律（非色彩）辨識，5 人測試 ≥ 70%
- [ ] Reduce-Motion 模式啟用後：素材飛入動畫停用；難度卡片無進場動畫；計數器直接跳數
- [ ] 無障礙設定從主選單一級選項可達（不需多層進入）
