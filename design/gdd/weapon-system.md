## A. 概覽 (Overview)

武器系統是殲獸戰機核心狩獵循環的輸出端。玩家同時裝備一把**主武器**（雷射系 / Laser，4 選 1）與一把**副武器**（飛彈系 / Missile，4 選 1），透過「雙軌核心機制」對巨獸部位造成破壞：雷射系對部位施加**蓄熱**（Heat），使部位進入**軟化**（Softened）狀態；飛彈系對部位施加**破甲**（Armor-Break），在部位軟化後大幅提升效率，最終達成**部位破壞**（Part Break）。八種武器圍繞同一功率預算 D₀ 設計——任何武器的最優持續輸出等量，差異僅在於輸出如何分配至不同戰術軸（DPS、覆蓋範圍、追蹤、破甲效率、持續火力比例），徹底落實「橫向選擇」支柱。每種武器的 Tier-3 強化引入獨特機制，深化而非數值放大其身份定位。

## B. 玩家幻想 (Player Fantasy)

**目標 MDA 美學**：挑戰（Challenge）＋表達（Expression）＋發現（Discovery）

玩家感受的是「科學家與獵人合一」的雙重幻想：

**「我看穿了你」** — 每隻巨獸的部位構造暗示了哪種雷射最快讓它軟化、哪種飛彈最乾脆讓它爆裂。讀懂怪物等於選對 loadout。正確組合讓戰鬥從「硬打」變成「精準拆解」，勝利感來自洞察而非蠻力，呼應概念文件的核心幻想：「以小搏大、以智取勝」。

**「這把武器是我的」** — 八種等值武器沒有唯一解。玩家可以堅守自己的打法身份（例如永遠用集束雷射×追蹤飛彈的「精準射手」流派），也可以在特定關卡換裝（例如對蛇形巨獸改用穿透雷射）。這是身份認同的選擇，不是效率最優化的計算，直接服務「橫向選擇」支柱。

**「那一聲爆破」** — 部位軟化到位、魚雷命中、熱衝擊引爆觸發的瞬間是系統給予的最高爽點。它需要蓄熱準備、精確射擊、時機判斷三者到位——爽感是被賺到的，直接服務「破壞即獎勵」支柱。

## C. 詳細規則 (Detailed Rules)

### C.1 雙武器池架構

| 武器池 | 類型 | 數量 | 機制角色 |
|--------|------|------|----------|
| 主武器池（Primary Pool） | 雷射系（Laser） | 4 種 | 對部位施加蓄熱，使其進入軟化狀態 |
| 副武器池（Secondary Pool） | 飛彈系（Missile） | 4 種 | 對部位施加破甲，軟化後效率大增 |

**裝備規則**：
- 玩家同時持有且只持有 1 把主武器 + 1 把副武器，共 8×8 = 64 種 loadout 組合。
- 兩池不互通：主武器插槽只能裝雷射系，副武器插槽只能裝飛彈系。
- 武器在戰鬥中透過**場地掉落**（Field Drop）取得，拾取即換裝，立即生效。
- 玩家的初始 loadout 由本輪進入前的永久選擇決定（詳見 F.3）。
- 玩家可刻意迴避武器莢艙以保留當前武器；無庫存系統，拾取即替換。

### C.2 雙軌核心機制：蓄熱→破甲（Soften → Break）

這是全系統的互動引擎，所有武器的設計都圍繞此機制構建。

#### 蓄熱階段

- 每個可破壞部位（Breakable Part）有一個**熱量槽**（Heat Gauge），範圍 0 到 H_max（預設 100 HU）。
- 雷射命中部位時，熱量槽以該武器的**熱量填充率**（H_rate，單位 HU/s）上升。
- 玩家停止以雷射命中部位時，熱量槽以**熱量衰減率**（H_decay_rate，預設 3 HU/s）自然冷卻。
- 有雷射命中時，衰減暫停（取最大值：`H_fill - H_decay = 淨上升`）。
- 熱量槽達到或超過**軟化閾值**（θ_S，預設等於 H_max）時，部位進入**軟化**（SOFTENED）狀態。
- 軟化狀態持續至部位被破壞，或熱量槽冷卻至**軟化退出閾值**（θ_S_exit，預設 80 HU）以下。
- 軟化狀態觸發明確的視覺/音效提示（紅橙色發光脈動 + 特定音效），且飛彈破甲效率提升（見下）。

#### 破甲階段

- 每個可破壞部位有一個**破甲槽**（Armor-Break Gauge），範圍 0 到 B_max（預設 100 BU）。
- 飛彈命中時，破甲槽以該武器的基礎破甲值乘以狀態乘數上升：
  - 部位**未軟化**（NORMAL）：× B_unsoftened_mult（預設 0.35）
  - 部位**已軟化**（SOFTENED）：× 1.0
  - 部位**震盪硬直**（STAGGERED，L3 Wave Cannon 震波觸發）：× 1.5（可與 SOFTENED 並存，不疊乘）
- 破甲槽達到 B_max（100 BU）時，部位發生**破壞**（Part Break），觸發視聽爆發 + 素材掉落。

**互鎖設計意圖**：未軟化部位的飛彈效率僅 35%，確保玩家不能跳過蓄熱步驟靠純飛彈暴力破甲。M3 穿甲魚雷（AP Torpedo）的熱衝擊引爆能力更進一步：觸發條件為部位已達軟化狀態，徹底鎖死「無蓄熱高傷害」路徑。詳見 E.1。

### C.3 武器取得

- 小怪擊殺或特定地形物件有機率掉落**武器莢艙**（Weapon Pod）。
- 武器莢艙掉落時明確標示類型（主/副）與武器名稱，玩家機體接觸即自動拾取。
- 拾取時直接換裝同類型武器，前一把丟棄（無庫存）。
- 武器莢艙的掉落池由當前關卡設定決定（詳見 F.2 掉落系統）。

## D. 公式 (Formulas)

### D.1 D₀ 功率預算模型

**定義**：D₀ 是全系統功率基準單位，表示一把武器在其完整彈匣＋換彈週期內的**持續等效輸出**。單位：功率單位/秒（Power Unit per second，PU/s）。

**等功率約束（Equal Power Constraint）**：

```
Sustained_Output(weapon) = D₀  ±  10%
```

**持續輸出計算**：

```
Sustained_Output = Total_Output_per_Mag / (Mag_Duration + Reload_Time)
```

變數定義：
- `Total_Output_per_Mag`：一整個彈匣在最優命中條件下的總輸出（PU）
- `Mag_Duration`：打完一整個彈匣所需時間（秒）
- `Reload_Time`：換彈時間（秒）

**等功率確認範例 — M3 穿甲魚雷（AP Torpedo）**：

```
Total_Output_per_Mag = 3 枚 × 3×D₀（未軟化基礎值）= 9×D₀
Mag_Duration ≈ 3s（3 枚間隔）
Reload_Time = 4s
Sustained_Output = 9×D₀ / 7s ≈ 1.29×D₀
```

> M3 持續輸出略高於基準，以補償其無追蹤、中速的命中難度。熱衝擊引爆（6×D₀）是情境爆發值，非持續值；實際命中率修正後整體在 ±10% 容差內。最終數值見 G 章節調校旋鈕。

**單位換算錨點（Unit Conversion Anchor）**：

系統存在三種計量單位，必須以固定換算比綁定，否則「×D₀」的武器數值無法與破甲/蓄熱條的 BU/HU 數值互相驗證（此為自動化平衡測試 H.1–H.3 的先決條件）：

```
1 D₀ 的持續輸出  ≙  10 BU/s   （對已軟化部位的破甲填充速率）
即：  破甲值(BU) = 武器輸出(×D₀) × 10
```

- 驗證：M3 未軟化單發 = 3×D₀ → 瞬間破甲 30 BU；經 `B_unsoftened_mult`(0.35) 修正 = **10.5 BU**（與 D.3 表一致）。
- 蓄熱單位 HU 屬**軟化軌**，與破甲軌 BU 各自獨立校準（見 G 章 `H_max_*` / `B_max_*`），兩者不共用此換算；D₀→BU 換算僅適用於破甲軌。

### D.2 蓄熱系統公式

**熱量更新（每幀）**：

```
H(t) = clamp( H(t-1) + H_fill(t) - H_decay(t),  0,  H_max )

H_fill(t)  = H_rate_weapon × Δt      （雷射命中時）/ 0（未命中時）
H_decay(t) = H_decay_rate × Δt       （雷射未命中時）/ 0（命中時）
```

**軟化狀態判定**：

```
if H(t) >= θ_S:
    part.state = SOFTENED

elif part.state == SOFTENED and H(t) < θ_S_exit:
    part.state = NORMAL
```

**各雷射 H_rate 預設值**：

| 武器 | H_rate（最優命中） | 備注 |
|------|-------------------|------|
| L1 散波雷射（三束全中） | 25 HU/s | 三束分散注意力但多部位同時蓄熱 |
| L1 散波雷射（僅中央束） | 8.3 HU/s | 精準度下降時大幅降低 |
| L2 集束雷射 | 37.5 HU/s | 單部位最快蓄熱 |
| L3 波動砲（短脈衝連發） | 15 HU/s | 主場景為剝甲，非蓄熱主力 |
| L3 波動砲（蓄力震波，Tier-3） | 50 HU 即時注入 | 每次蓄力週期（1.5s + 2s CD）的熱量脈衝 |
| L4 穿透雷射（每部位） | 25 HU/s | 路徑上各部位各自獲得完整 H_rate |

**軟化時間（T_soften）推導**：

```
T_soften = θ_S / max(H_rate - H_decay_rate,  ε)
```

（ε = 0.01，防止除零；若 H_rate ≤ H_decay_rate 則部位永不軟化——設計時須避免）

> **範例**：θ_S = 100，H_decay_rate = 3 HU/s，L2 集束雷射 H_rate = 37.5 HU/s
> T_soften = 100 / (37.5 - 3) ≈ **2.9 秒**（理論最短；實戰預計 5–8 秒）

### D.3 破甲系統公式

**破甲更新（每次飛彈命中事件）**：

```
B_fill = B_rate_weapon × M_state_mult
B(new) = clamp( B(current) + B_fill,  0,  B_max )
```

**M_state_mult 查表**：

| 部位狀態 | M_state_mult |
|----------|-------------|
| NORMAL（未軟化） | B_unsoftened_mult（預設 0.35） |
| SOFTENED（已軟化） | 1.0 |
| STAGGERED（震盪硬直） | 1.5 |
| SOFTENED + STAGGERED | 1.5（不額外疊乘） |

**各飛彈 B_rate_weapon 預設值（軟化條件，即 M_state_mult = 1.0）**：

| 武器 | 每次觸發 B_fill | 備注 |
|------|----------------|------|
| M1 追蹤飛彈（2 枚全中） | 33.3 BU/s 有效持續 | 命中率由追蹤保障 |
| M2 蜂群飛彈（8 枚全中，單目標） | 25 BU/s 有效持續 | 多目標時各目標分配 D₀/8 |
| M3 穿甲魚雷（軟化，熱衝擊引爆） | 60 BU 即時（= 6×D₀ × 10） | 1 枚填充 60%，2 枚破壞標準部位 |
| M3 穿甲魚雷（未軟化） | 10.5 BU（= 30×0.35） | 低效路徑；確保跳過蓄熱有懲罰 |
| M4 叢集炸彈（N=1） | 20 BU/hit | N=1 時最高效率（2×D₀） |

**部位破壞判定**：

```
if B(new) >= B_max:
    trigger PART_BREAK(part_id)
    reset B(part_id) = 0
```

### D.4 TTB（Time-To-Break，部位破壞時間）公式

TTB 是單一部位從完整護甲到破壞所需的總時間，作為平衡性驗證的主要指標。

```
TTB = T_soften + T_break

T_soften = θ_S / max(H_rate - H_decay_rate, ε)

T_break  = B_max / B_rate_softened           （一般飛彈）
T_break  = ceil(B_max / B_fill_per_shot) × shot_interval   （M3 等離散命中）
```

**TTB 設計目標範圍**（預設護甲，一般難度）：

| 部位類型 | TTB 目標 | 說明 |
|----------|----------|------|
| 普通部位（Normal Part） | 15–25 秒 | 主要戰鬥節拍 |
| 強化部位（Armored Part） | 30–45 秒 | 需要良好 loadout 配合 |
| Boss 核心部位（Core Part） | 50–80 秒 | 多階段高潮 |

**TTB 範例計算 — 搭配 B（L2 集束雷射 × M1 追蹤飛彈，普通部位）**：

```
T_soften = 100 / (37.5 - 3) ≈ 2.9s（理論）；實戰約 6s
T_break  = 100 / 33.3 ≈ 3.0s（軟化後）
TTB ≈ 9s 理論 / 約 15s 實戰
```

**TTB 範例計算 — 搭配 A（L1 散波雷射 × M3 穿甲魚雷，普通部位）**：

```
T_soften = 100 / (25 - 3) ≈ 4.5s（理論）；實戰約 8s
T_break  = 需 2 枚魚雷（各 60 BU）→ 約 4s（含射擊間隔）
TTB ≈ 8.5s 理論 / 約 12s 實戰
```

### D.5 8×8 矩陣等功率驗證指引

每次修改武器數值後執行：

```
For each Primary P in {L1, L2, L3, L4}:
  For each Secondary S in {M1, M2, M3, M4}:
    Assert Sustained_Output(P) ∈ [0.9×D₀, 1.1×D₀]
    Assert Sustained_Output(S) ∈ [0.9×D₀, 1.1×D₀]
    For each PartTier in {Normal, Armored, Core}:
      Assert TTB(P, S, PartTier) ∈ [TTB_min(PartTier), TTB_max(PartTier)]
      Assert TTB(P, S, PartTier) <= 2.0 × min_TTB(PartTier)
```

## E. 邊界情況 (Edge Cases)

### E.1 [極高風險] M3 熱衝擊引爆是否跳過必要行動

**情況**：6×D₀ 爆發是否因為太強，使「蓄熱」步驟對 M3 玩家變成可跳過的障礙，導致其他所有武器組合相對失色？

**設計防護**：
1. 熱衝擊引爆觸發條件為 `part.state == SOFTENED`（H ≥ θ_S = 100%），無例外路徑。
2. 未軟化時 M3 破甲效率：3×D₀ × 0.35 = 1.05×D₀——等同最弱一般飛彈，無爆發。
3. 部位設定的護甲容量（B_max）需使「跳過蓄熱暴力破甲」的 TTB ≥ 正常路徑的 1.5×。

**待驗證**：原型階段對各護甲數值測量 TTB 矩陣，確認任何 loadout 組合下跳過蓄熱的路徑均不可行。

**若原型顯示問題**：調整 `B_unsoftened_mult`（降至 0.2–0.25），或為每部位加入**熱衝擊引爆冷卻**（CD = 8 秒，防止連續引爆）。

### E.2 熱量衰減速度失衡

- **衰減過快**：玩家無法維持軟化狀態 → 雙軌機制失效。對策：H_decay_rate ≤ 最慢 H_rate 的 15%（預設 3 HU/s ≤ 25 × 15% = 3.75）。
- **衰減過慢**：一次蓄熱後飛彈可無限輸出多輪，蓄熱工作失去挑戰性。對策：設 θ_S_exit = 80%；高壓彈幕迫使玩家閃避時，熱量仍緩慢冷卻。

### E.3 L4 穿透雷射的關卡依賴性

若某 Boss 部位均橫向排列，L4 的垂直穿透優勢完全喪失，等功率約束下它淪為「最差選項」。見 F.5 與 H.6：關卡設計師須在每個 Boss 保證至少一組垂直部位排列。

**備案（若後期發現問題）**：為 L4 加入第 5 發切換水平方向的穿透，確保任何情境下的基礎可用性。

### E.4 L3 波動砲弱點窗口長度

- **< 1.5s**：C 組合（震盪-飽和）中玩家無法在窗口內完成蜂群飛彈齊射，組合失效。
- **> 3s**：窗口過舒適，「高天花板」定位消失。
- **預設 2.0s**：原型從 1.5s 起步，以「玩家能否在窗口內完成一次 M2 齊射」為錨點調整。

### E.5 M4 叢集炸彈 N=1 的 2×D₀ 輸出

當 AoE 範圍內只有 1 個部位時，單體受 2×D₀ 破甲。不構成平衡問題：M4 落點固定在前方特定區域，玩家無法任意選擇目標；N=1 屬 Boss 頂部孤立部位的罕見情況。保留此數值作為 M4 的「意外高效」時刻。

### E.6 多種殘熱效果疊加（L1 Tier-3 殘熱焰 + L4 Tier-3 熱殘影）

若兩者同時作用於同一部位，取**最高 H_rate 值**，不相加（防止 Tier-3 組合蓄熱爆表）：

```
H_residual_fill(t) = max(l1_residual_rate, l4_afterimage_rate) × Δt
```

主動雷射命中時的 H_fill 獨立計算，不受此規則約束。

### E.7 武器彈匣耗盡

- M1 耗盡：換彈 3 秒，期間可持續使用主武器。
- M3 耗盡：輸出歸零 4 秒——這是 M3 作為「高賭注武器」的刻意設計代價。
- 拾取同型武器莢艙：彈匣補滿，換彈進度重置。

### E.8 破壞後部位狀態

部位破壞後：碰撞體從 Boss 判定移除（切換已破壞動畫態）；熱量槽與破甲槽歸零；繼續射擊該位置無任何效果（穿透，空白音效回饋）。

## F. 系統相依 (Dependencies)

### F.1 可破壞部位系統（Breakable Part System）——核心依賴

武器系統**輸出**給此系統：
- `on_laser_hit(part_id, H_fill_amount)` — 熱量填充事件
- `on_missile_hit(part_id, B_fill_amount, state_mult)` — 破甲填充事件
- `on_l3_wave_hit(part_id, kaiju_id)` — L3 波動砲蓄力震波命中（觸發震盪硬直／剝甲；由 kaiju-part-system 消費）
- `on_part_break(part_id)` — **由部位系統發出**、武器系統接收以清除碰撞體並執行 L2 Tier-3 漣漪；此處列出僅表雙向資料流（權威定義見 kaiju-part-system.md）

> **附錄（非破壞性補充，2026-07-01）**：`on_l3_wave_hit` 為 L3 波動砲的第三條輸出事件通道，原 F.1 未列出，於此補入。文件主體數值仍 LOCKED，此為介面對齊補充。

武器系統**從此系統接收**：
- 每個部位的當前狀態（NORMAL / SOFTENED / STAGGERED / BROKEN）
- 部位的 H_max、B_max（由巨獸定義檔決定）
- 部位的世界座標（供追蹤飛彈鎖定）

### F.2 掉落系統（Drop System）

武器系統消費此系統的輸出（武器莢艙）。掉落池由關卡設定決定，武器系統不控制掉落頻率。

### F.3 永久強化系統（Permanent Upgrade System）

- 武器的 Tier 等級由此系統控制，影響讀取的調校旋鈕數值（Tier-3 觸發各武器的獨特機制）。
- 武器系統不直接操作素材庫存；素材由 `on_part_break` 事件傳遞給材料系統。

### F.4 難度系統（Difficulty System）

- 難度主要影響敵彈密度，間接影響玩家有效蓄熱 uptime（高難度下玩家需閃避，命中率下降）。
- 武器數值本身不依難度縮放——高難度下 TTB 自然上升，是隱性難度調節。

### F.5 關卡設計（Level Design）

- **L4 可用性依賴**：關卡設計師須在每個 Boss 保證至少 1 組「垂直對齊部位」（≥2 個）。
- **M4 落點依賴**：Boss 應有「頂部密集部位」的設計語言以發揮 M4 優勢。
- 每個 Boss 設計評審須附上武器可用性矩陣確認（至少每種武器有 1 個最佳情境的部位設計）。

### F.6 輸入系統（Input System）

- L3 蓄力模式：需「長按計時事件」（長按主武器鍵 ≥ l3_charge_time 秒觸發）。
- M4 落點指示器：視覺元素由 UX 系統繪製，武器系統提供落點座標。
- 觸控平台風險：長按判定可能與移動滑動衝突（見 I 開放問題）。

### F.7 視覺/音效回饋系統（VFX / SFX System）

- **軟化狀態提示**（紅橙色發光脈動 + 音效）是雙軌機制**可讀性的核心**——見 H.5 驗收標準，此為 UX 阻斷級條件。
- 部位破壞的視聽爆發服務「破壞即獎勵」支柱。
- 各武器命中音效需對應性格（雷射系：乾脆高頻；飛彈系：衝擊低頻）。

## H. 驗收標準 (Acceptance Criteria)

### H.1 等功率等價（功能性 — 阻斷）

- [ ] 任何單一武器在最優命中條件下，持續 30 秒的總輸出位於 `D₀ × [0.9, 1.1]` 之間。
- [ ] 自動化測試：`tests/unit/weapon/weapon_dps_equivalence_test.[ext]`，每次數值改動後自動運行。

### H.2 無主導 loadout（功能性 — 阻斷）

- [ ] 在「標準 Boss（普通部位 ×3）」場景下，8×8 = 64 組 loadout 的 TTB 中，任何 loadout 的 TTB 不超過最快 loadout 的 2.0× 以上。
- [ ] 無任何 loadout 在全部三種部位類型（普通/強化/Boss 核心）上均排名前三。
- [ ] 驗證工具：`tests/unit/weapon/weapon_loadout_matrix_test.[ext]` 輸出 8×8 TTB 矩陣供設計師閱讀。

### H.3 M3 熱衝擊引爆護甲門檻（功能性 — 阻斷）

- [ ] 在任何合法護甲設定下，「跳過蓄熱直接以 M3 暴力破甲」的 TTB ≥ 正常蓄熱路徑的 1.5×。
- [ ] 自動化測試：`tests/unit/weapon/m3_gate_validation_test.[ext]`，遍歷 B_unsoftened_mult 合法範圍驗證。

### H.4 L3 波動砲弱點窗口可操作性（體驗性）

- [ ] `l3_stagger_window = 2.0s` 設定下，玩家能在窗口內完整完成一次 M2 蜂群飛彈齊射（8 枚）。
- [ ] 測試方法：在 1.5s / 2.0s / 2.5s 三種設定下各做 10 次實際按鍵序列測試，記錄成功率。目標：2.0s 設定下 ≥ 80% 成功率。

### H.5 雙軌可讀性（體驗性 — UX 阻斷）

- [ ] **「部位軟化」視覺提示在 0.5 秒內讀得懂**：不熟悉遊戲的受測者能在 10 張靜態截圖中正確辨識軟化部位，成功率 ≥ 80%。
- [ ] 驗收方法：設計師主持 5 人用戶測試，展示含不同彈幕密度的截圖。
- [ ] 若未達標：UX 提升行動（更大光暈、更強對比顏色、脈動動畫）升為最高優先級，**阻斷 Alpha 里程碑**。

### H.6 L4 關卡可用性（功能性）

- [ ] 每個 Boss 設計中，至少存在 1 個「穿透雷射最優情境」（≥2 個垂直對齊的可破壞部位）。
- [ ] 由關卡設計師在 Boss 設計評審時確認，記錄於對應 Boss 設計文件。

### H.7 Tier-3 強化深化身份，不放大數值（功能性）

- [ ] 每種武器的 Tier-3 強化不使普通部位的 TTB 縮短超過 15%（相對 Tier-1 基準）。
- [ ] 自動化測試：`tests/unit/weapon/tier3_identity_depth_test.[ext]`，對比 Tier-1 vs Tier-3 的 TTB。

### H.8 標誌性搭配節奏感差異（體驗性）

- [ ] 三組搭配（A 軟化-收割、B 專注-放心、C 震盪-飽和）各完成 10 次原型 Boss 戰後，主要設計師能清晰描述其打法節奏感的差異。
- [ ] 若兩組搭配在描述上無法區分，視為差異化不足，需回頭調整對應武器參數。
