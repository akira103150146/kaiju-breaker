# 導演決議 — 2026-07-03（7 點意見設計規格審閱回覆）

> 導演把回覆打在各設計文件的 Obsidian 副本的開放問題後面；本文件把全部決定彙整成 repo 的權威記錄，供實作依據。原文件：`design/gdd/*`, `design/art/*`, `design/narrative/*`, `design/quick-specs/*`。

## 兩個總原則（最高優先）
1. **架構彈性優先**：設計要讓後期新增關卡/怪物/彈幕種類都能**很快加**（資料驅動、SO 授權、可組合）。→ 記憶 `extensibility-first-fast-content-add`。
2. **還不是正式版，別糾結平衡**：機制做對、數字可調就好，之後大量微調。→ 記憶 `tunable-first-formulas-matter`。

## 點 2 — 玩家射向（集氣改直向）
1. L3 縱向震波形狀 → **往上的光柱**（非扇形）。
2. 左右微瞄 → **完全固定 +Y**；玩家自己移動戰機瞄準（技術性）。
3. per-level「射向基準」覆寫欄位 → **先預留**。

## 點 3 — 武器分級 + 等功率
- L3 T3「共鳴擴散」+80% 超標 → **選 C**：定位為情境爆發值（比照 M3 熱衝擊），**不受 H.1/H.7 穩態約束**。
- M1/M4 `ShotInterval` → **先用 path-A 快速值**（M1 0.10s；M4 0.20s；導演記「先用0.1s」= 用最快 path-A 佔位，之後原型微調）。
- M4 `m4_t3_child_dmg_pct` → **直接收緊 0.18**。
- L2/M3 `EffectiveHitRate`（0.65 / 0.80）→ **先不用實測驗證**，沿用分析值。
- M2 T3「飽和點名」查詢介面 → **在 `IPartStateQuery` 補一個 `GetHottestAlivePartId` 的姊妹方法**（取 BU 進度最高的已軟化部位）。
- L1 波束階梯視覺成長 → **先用簡單 sprite scale 佔位**。
- `EnemyBulletSizeConfig.ThreatWeight` 消費端 → **等點 1 彈幕多元化正式立項時一併設計**。

## 點 7 — 怪物數值/機制分級
1. `PartType.MidCore` → **新增**（+ `on_mid_core_break` 事件；改 `kaiju-part-system.md`，跨系統）。
2. 破部位→AI/移動反應（破腿降速）契約 → **新增一份專門處理怪物移動模式的規格**。
3. 雜魚 HP 校準 → **雜魚 = 最基本武器打 2 發可擊殺**（作為 HP placeholder 的錨）。
4. Mid/Boss 部位數指引重疊 → **可接受**（以事件連結判別，非部位數）。
5. `shield_flier` 遷移為 `MechanicPatternSO` → **可以（遷移）**。

## 點 6 — 打擊感分級
1. 最終頭目子分級 → **先不用擴充**（全 Boss 一視同仁）。
2. Gate 破除回饋 → **獨立設計**（不沿用護甲剝除）。
3. 雜兵/菁英 SFX 資產 → **先用既有音效調整頂上**。
4. `kill_feel_simultaneous_budget` 隨難度 → **是，要隨 D1–D4 調整**（新增 difficulty 交叉點）。
5. 波段清空收束 → **做**：一波清完後閃「CLEAR」字樣（古早橫版清關風）。

## 點 4 — 背景捲動

2. 反向捲動 → **不需要**。
3. 近景層/分層 parallax → **先不做分層**（先簡單單層背景）。
4. `bg_segment_variant_swap`（貼圖變體切換）→ **做**。
5. 交叉引用 `art-bible.md`/`stage-system.md` → **核准**。

2. 五區主題/順序 → **先這樣五關**，但**保留後期快速新增更多關卡的預設**（呼應總原則 1）。VOLTWYRM 移第四區。
